Design

  • Input & Output (Axel) - A textarea was chosen in order to allow paragraph inputs to be easily viewed, and the output is constrained to a box for sheer visual effect.
  • Styling (Axel) - I used my previously created HTML & CSS template for this project. Some small updates were necessary to make things look good and stay intact.
  • Diacritical Marks and Formatting (Axel) - I borrowed inspiration for the diacritical marks from Tchouk's Zalgo Text Generator. I had this mix with coloration and font-size, each completely randomizing per call. Thus, the output is "mutilated" such that it will have combining marks glued to each character, as well as each character varying in size and color.
  • Content Corruption (Will) - Each character from the input is replaced with some string based on character. Certain replacements have random characters mixed in.

Challenges

Learning (Will)

The primary challenge here was actually just trying to learn the intricacies of Javascript as compared to Java. Personally, I found it difficult to sort out the minutae of the project -- Is that supposed to be a double quote, or a single quote? How do you interface between JS and HTML? -- regardless, the challenges weren't too various, it was mostly just a lot of legwork, putting in the different outputs.

Random Decimals to Hex numbers (Axel)

I spent quite a long time trying to search for ways on how to randomize hex values such that I could use them for the Unicode Diacritical Marks and the font colors.

Troubleshooting (Axel)

Javascript is excellent at not being clear about errors half the time. Sometimes the error is clearly explained. Sometimes I need to check Stack Overflow.
