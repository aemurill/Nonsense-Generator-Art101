Design

  • Input & Output (Axel) - A textarea was chosen in order to allow paragraph inputs to be easily viewed, and the output is constrained to a box for sheer visual effect.
  • Styling (Axel) - I used my previously created HTML & CSS template for this project. Some small updates were necessary to make things look good and stay intact.
  • Diacritical Marks and Formatting (Axel) - I borrowed inspiration for the diacritical marks from Tchouk's Zalgo Text Generator. I had this mix with coloration and font-size, each completely randomizing per call. Thus, the output is "mutilated" such that it will have combining marks glued to each character, as well as each character varying in size and color.
  • Content Corruption (Will) - Each character from the input is replaced with some string based on character. Certain replacements have random characters mixed in.
  • Language Translation (Axel) - the string is sent to Google's translation service and returned. This uses an improper AJAX method, but works very well. GOOGLE DOES OCCASIONALLY REFUSE TO SERVICE REQUESTS, BUT THIS IS PART OF NOT HAVING PAID 75$ PER HOUR
  • Toggle-able Interface (Axel) - All the mutilations can be turned on and off!. Languages can also be selected!
  • MutilationObject (Axel) - the string was turned into an object to take advantage of the properties of an object, as well as to make using functions (now methods) even more simple than before. Plus, being able to keep all the relevant data within an object made keeping track of changes easier.
  • Message from Will - Comparatively few design changes made by me are present in this final iteration. Axel handled the brunt of the finalization process, but collaborative design endeavors proved fruitful. Input in the form of speculative design bridged the gap between the midterm iteration of this project and its current iteration.

Challenges

Learning (Will)

The primary challenge here was actually just trying to learn the intricacies of Javascript as compared to Java. Personally, I found it difficult to sort out the minutae of the project -- Is that supposed to be a double quote, or a single quote? How do you interface between JS and HTML? -- regardless, the challenges weren't too various, it was mostly just a lot of legwork, putting in the different outputs.

Random Decimals to Hex numbers (Axel)

I spent quite a long time trying to search for ways on how to randomize hex values such that I could use them for the Unicode Diacritical Marks and the font colors.

Troubleshooting (Axel)

Javascript is excellent at not being clear about errors half the time. Sometimes the error is clearly explained. Sometimes I need to check Stack Overflow.

React-ability (Axel)

Making the JS respond to inputs, and having the page react to JS changes was key to making everything work. I took much longer than expected in trying to streamline the process, as well as having the page work properly after accidentally breaking the checkbox updating system. I ended up needing to create Enums, and various data objects that contained values needed for not needing to repeat myself.

AJAX and callbacks (Axel)

I hate callbacks. At least, I hate how I was seemingly unable to get the system to wait for the AJAX event to finish before returning a value, rather than just having it call another function. I did this because I did not want to have to break up my flow into chunks just for method called by a function which was just a fraction of the process. As result, I decided to use a SYNCHRONOUS AJAX call, which is high heresy but was required for the functionality I was going for. That and my attempts to make the system wait for an async value kept NOT waiting for the value.

Objectifying the String (Axel & Will)

Turning the mutilated string into an object was tricky in that it required understanding how we were going to use our object. It was ultimately decided that having a MutilationObject contain the current data as well as have methods would be better than simply passing an object to a bunch of unrelated functions.

A Message from Will

Wrapping my head around APIs, and understanding the implementation of both the requirements asked of us from Wes and objectives that our team set forth for the project. The ultimate challenge of continuing this project was trying to continue the vision we had for the midterm, and exploring where we could and should take the project form there. Pondering how we could go about adding features was a bit of a challenge, too, as it was sometimes difficult to communicate exactly what form our project would end up taking..
